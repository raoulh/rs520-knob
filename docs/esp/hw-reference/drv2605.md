# DRV2605 Haptic Driver — Hardware Notes

App-facing reference for the TI DRV2605 haptic motor driver.

## Overview

- Single-channel haptic driver for ERM (eccentric rotating mass) and LRA (linear resonant actuator) motors
- I2C control with on-chip 123-effect waveform library
- Used for volume-step clicks, button-press feedback, transport confirmations

## Electrical

| Parameter | Value |
|-----------|-------|
| Supply | 2.0–5.5V |
| I2C Address | 0x5A (7-bit) |
| I2C Speed | 100–400 kHz |
| SDA | GPIO 11 (shared with CST816) |
| SCL | GPIO 12 (shared with CST816) |
| Output | H-bridge to ERM/LRA motor |

## Actuator Setup

1. Select actuator type (ERM or LRA) in control/feedback registers
2. Set Rated Voltage and Overdrive Clamp registers
3. Run Auto-Calibration on first boot:
   - Set Auto-Cal mode, GO=1
   - Wait for GO=0
   - Read status for success/fail
   - Store calibration results in NVS for subsequent boots

For LRA: calibration also measures resonance frequency.

## Basic Playback (Internal Library)

```
1. Ensure device is out of standby (Mode register)
2. Select waveform library (ERM or LRA)
3. Program Waveform Sequence slots (slot0..slotN) with effect IDs; terminate with 0x00
4. Set GO=1 → driver plays sequence → GO returns to 0
```

### Recommended Effects for RS520-Knob

| Action | Effect | Notes |
|--------|--------|-------|
| Volume step (encoder detent) | Strong Click (short, crisp) | Per-detent feedback |
| Play/pause tap | Sharp Click | Slightly different feel |
| Transport (next/prev) | Soft Bump | Lighter |
| Zone switch | Double Click | Confirmation |

## Real-Time Playback (RTP)

For custom feel without library IDs:
- Select RTP mode
- Write amplitude to RTP register at ~200–300 Hz while GO is set
- Useful for "buzz while pressed" effects

## App Integration

```cpp
// Init once at boot
void haptic_init() {
    // Bring out of standby
    drv2605_write_reg(MODE_REG, 0x00);  // Internal trigger

    // Auto-cal on first boot, or load from NVS
    if (!load_calibration_from_nvs()) {
        run_auto_calibration();
        save_calibration_to_nvs();
    }

    // Select library
    drv2605_write_reg(LIBRARY_REG, 0x01);  // ERM library
}

// Play a click
void haptic_click() {
    drv2605_write_reg(WAVEFORM_SEQ_0, 1);  // "Strong Click"
    drv2605_write_reg(WAVEFORM_SEQ_1, 0);  // End
    drv2605_write_reg(GO_REG, 1);
}
```

## Haptics Queue

Keep a non-blocking queue so UI events don't overlap pulses. Drop or coalesce fast repeats (e.g., rapid encoder rotation).

## Gotchas

- **Don't start new effect while GO=1** — queue or ignore until it returns to 0
- **Rated/Clamp voltage too high** can over-drive motor — start conservative, calibrate
- **LRA effects feel weak?** Switch libraries and re-run calibration; resonance matters
- **I2C bus shared with CST816** — ensure mutual exclusion (mutex)
- **Motor current transients** can glitch the I2C bus — ensure decoupling near driver
